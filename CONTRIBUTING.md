# Посібник для контриб'юторів (Contributing Guide)

Дякуємо за інтерес до проєкту **Notary Document Generator**! Ми вітаємо будь-яку допомогу: від виправлення багів до додавання нових шаблонів та функцій.

Цей документ описує, як саме ви можете розширювати функціонал програми без порушення її архітектури.

## Як додати нове поле у документ

Архітектура програми побудована таким чином, що функція генерації документа (`replace_placeholder_in_document`) є універсальною. Вона автоматично обробляє всі дані, передані їй у словнику. 

Щоб додати нове змінне поле в існуючий документ, вам **не потрібно** змінювати логіку роботи з файлами `.docx`. Достатньо виконати 4 прості кроки:

### Крок 1. Додайте мітку в шаблон Word
Відкрийте файл `Templates.docx` та вставте новий плейсхолдер у потрібне місце. Використовуйте подвійні фігурні дужки великими літерами, наприклад: `{{НОВЕ_ПОЛЕ}}`.

### Крок 2. Зареєструйте змінну в коді
Відкрийте файл `placeholders.py` і додайте константу, щоб уникнути одруківок при використанні в інших файлах:
```python
NEW_FIELD = '{{НОВЕ_ПОЛЕ}}'
```
### Крок 3. Створіть поле вводу (GUI)
У файлі потрібного екрану (наприклад, `Zayava_Form_1_batko.py`) створіть віджети для вводу даних за допомогою tkinter.ttk:
```python
label_new = ttk.Label(document_form_frame, text="Назва нового поля:")
entry_new = ttk.Entry(document_form_frame, width=50)

# Порада: Не забудьте додати нове поле у словники валідації 
# (entries_to_validate, validators_map), якщо воно потребує перевірки!
```
### Крок 4. Передайте дані у генератор
Знайдіть функцію-обробник кнопки генерації (наприклад, `on_button_click())`. Зчитайте текст із вашого нового поля і збережіть його у головний словник `app_data`
```python
# Зчитуємо дані з інтерфейсу
new_value = entry_new.get()

# Записуємо в словник під ключем нашого плейсхолдера
app_data["user_data"][NEW_FIELD] = new_value
```
## Як зробити Pull Request
Якщо ви хочете запропонувати свої зміни до головного репозиторію, будь ласка, дотримуйтесь цього процесу:

Зробіть Fork цього репозиторію.

Створіть нову гілку для вашої фічі `(git checkout -b feature/AmazingFeature)`.

Закомітьте ваші зміни `(git commit -m 'feat: Add new amazing feature')`. Обов'язково пишіть зрозумілі повідомлення до комітів англійською мовою.

Відправте зміни у свій форк `(git push origin feature/AmazingFeature)`.

Відкрийте Pull Request.

Перед створенням PR переконайтеся, що ваш код не містить закоментованого сміття (мертвого коду) та відповідає загальному стилю написання (PEP 8).
