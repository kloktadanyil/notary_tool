Программа для автоматизації роботи нотаріуса. Цей додаток з графічним інтерфейсом (GUI) дозволяє швидко та безпомилково генерувати юридичні документи на основі попередньо створених шаблонів Word (наразі підтримується генерація "Заяви на виїзд дитини за кордон").

## Особливості проєкту

* **Графічний інтерфейс:** Покроковий "майстер" створення документа, побудований на бібліотеці `tkinter` та `ttk`.
* **Динамічна валідація:** Вбудована перевірка введених даних (регулярні вирази) для ПІБ, дат (ДД.ММ.РРРР), УНЗР, ідентифікаційних кодів (10 цифр) та типів паспортів з підсвічуванням помилок.
* **Генерація документа:** Збереження форматування оригінального `.docx` шаблону при заміні плейсхолдерів (наприклад, `{{ПІБ}}`, `{{ДАТА_НАРОДЖЕННЯ}}`) за допомогою бібліотеки `python-docx`.
* **Мовна адаптація (NLP-елементи):** Автоматичне схиляння українських слів, правильні закінчення залежно від статі (наприклад, "син/донька", "його/її") та конвертація дат і чисел у текст (порядкові та родові відмінки).
* **Розумні блоки:** Скрипт автоматично визначає тип паспорта користувача (старий зразка "книжечка" або нова ID-картка) і видаляє з фінального документа непотрібний текстовий блок.
* **Динамічні поля:** Можливість додавати необмежену кількість дітей та супроводжуючих осіб у заяву з автоматичним корегуванням граматики (наприклад, "моєї дитини" / "моїх дітей").

## Як працює архітектура програми

Додаток розділений на логічні модулі для зручної підтримки та масштабування:

1. **GUI та Навігація (`main_app.py`, `vybir_zayavy.py` тощо):** Головний файл ініціалізує вікно розміром 1200x1000 та керує відображенням різних екранів (`Frame`). Користувач проходить шлях від головного меню до вибору типу заяви та заповнення даних батьків і дітей.
2. **Валідація (`validators.py`):** Містить функцію `create_validator_with_message`, яка створює правила перевірки для полів вводу. Якщо користувач вводить неправильний УНЗР або букви замість цифр, лейбл статусу миттєво стає червоним та виводить повідомлення про помилку.
3. **Обробка даних та логіка (`document_logic.py`, `Zayava_Form_1_batko.py`):** Тут зібрані дані з інтерфейсу. Система аналізує стать та автоматично підставляє правильні закінчення у змінні (`ending1`, `ending2`, `zaymennyk`). Також тут працює функція `convert_digit_to_text_uk`, яка перетворює системні дати на український текст.
4. **Робота з шаблоном та генерація (`document_logic.py`, `placeholders.py`):** Програма відкриває файл `Templates.docx` та шукає спеціальні мітки (плейсхолдери), прописані у константах (наприклад, `{{МІСТО}}`, `{{КВАРТИРА}}`). Функція `replace_placeholder_in_document` проходить по всіх абзацах та таблицях, акуратно замінюючи мітки на введені дані без втрати стилів шрифту.
5. **Збереження результату:** Готовий документ автоматично зберігається у створену директорію `Згенеровані документи/Заяви/За кордон` під іменем, яке відповідає ПІБ заявника (наприклад, `Іванов_Іван_Іванович.docx`).
